# Screen Design Guide

> 모든 화면 생성 및 수정 작업은 반드시 `./docs/DESIGN-apple.md`를 먼저 참조한 뒤 진행한다.
> 화면의 색상, 간격, 폰트, 버튼 스타일, 카드 스타일, 입력폼 스타일, 레이아웃 규칙은 `DESIGN-apple.md`의 정의를 우선 적용한다.

## 1. 화면 설계 요청 프롬프트 (템플릿)

화면 설계를 요청할 때는 아래 형식을 따른다. 디자인은 `./docs/DESIGN-apple.md` 파일을 참조한다.

- **무엇을 만드는지**: 어떤 화면을 만드는지
- **어떻게 배치할지**: 화면에 들어갈 요소와 배치 순서
- **어떤 정보를 보여줄지**: 화면에 표시/입력되는 데이터 항목
- **누가 사용할지**: 화면을 사용하는 사용자(역할)

### 화면디자인 공통 규칙
- 지시하기 전에는 HTML을 만들지 않는다.
- 지시 후 화면 검증을 위해 React Native용 화면과 React 웹용 화면을 각각 1개씩 HTML로 만든다.
- 모바일은 세로형 레이아웃, 웹은 가로형으로 하되 반응형 웹으로 만든다.
- HTML은 `./demo` 폴더에 샘플로 만든다.

## 2. 화면 요청 내역

### 2.1 로그인 화면
- **무엇을 만드는지**: 로그인 화면을 만든다.
- **어떻게 배치할지**: ID와 Pass를 입력할 수 있어야 하고, 그 아래에 로그인 버튼과 회원가입도 있어야 한다.
- **어떤 정보를 보여줄지**: ID, Password
- **누가 사용할지**: 의사와 환자가 사용한다.

### 2.2 환자 목록 화면
- **무엇을 만드는지**: 로그인이 성공하면 의사가 환자 목록을 볼 수 있는 화면을 만든다.
- **어떻게 배치할지**: 환자 목록이 보이고, 상단에는 환자이름·성별로 검색하는 기능, 하단에는 환자의 목록을 표시한다. 환자가 로그인한 경우 목록화면을 거치지 않고 자신의 건강 상세화면으로 바로 이동한다.
- **어떤 정보를 보여줄지**: 환자의 사진, 이름, 성별, 생년월일 정보를 목록에 표시한다.
- **누가 사용할지**: 의사가 사용한다.
