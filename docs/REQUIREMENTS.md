# 요구사항 정의서 (REQUIREMENTS)

## 1. 회원 로그인
환자와 의사가 ID, Password로 인증한 뒤 로그인한다.
- 회원 테이블과 데이터는 제공됨

## 2. 회원 목록 조회
회원 목록 정보를 표시 및 검색한다.
- 환자는 자기 자신의 정보만 조회 가능해야 한다.
- 의사는 모든 환자의 정보를 조회할 수 있어야 한다.

## 3. 회원 상세 조회
회원의 ID, 이름, 생년월일, 성별, 보유질병정보, 메모 정보를 조회할 수 있어야 한다.
- 실시간으로 전송되는 심박, 혈당, 걸음수 정보를 그래프로 표시한다.
- 질병코드 테이블과 데이터는 제공됨

## 4. 실시간 건강정보 조회
아래 항목별 실시간 전송 데이터를 조회할 수 있어야 한다.

| 구분 | 전송 데이터 |
|---|---|
| 심박정보 | 회원ID, 측정일시, 심박수 |
| 혈압정보 | 회원ID, 측정일시, 수축기 혈압, 이완기 혈압, 혈압상태 |
| 체중정보 | 회원ID, 측정일시, 체중, BMI |
| 걸음수 | 회원ID, 전송일시, 당일누적 걸음수 |

- 건강정보 제공 서버가 별도로 존재하며, 웹소켓(WebSocket)으로 데이터를 제공한다.
  - 서버 주소: https://healthsim.iranglab.com

## 5. 알림 기능
심박, 혈압, 혈당 정보 수신 중 이상 데이터 감지 시 관리자에게 알림을 전송한다.
